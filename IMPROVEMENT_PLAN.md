## 🥇 **P1 (필수)** - 즉시 구현 필요

### 1. **학번 필드 추가** ⭐⭐⭐
**현황:** 현재 이름만으로 학생 구분 → 동명이인 이슈 발생

**개선안:**
```
현재: name (예: "김철수")
개선: name + student_id (예: "001-김철수" 또는 "김철수 (001번)")
```

**이점:**
- 동명이인 완벽 구분
- 학교 시스템과 동기화 용이
- 보고서 작성 시 정확성 ↑

**구현:**
- Students 테이블에 `student_number` 필드 추가 (예: "001", "A-1" 등)
- UI에서 "이름 (학번)" 형식으로 표시
- 드롭다운, 리스트 모두 학번 포함 표시

---

### 2. **엑셀 파일 업로드 기능** ⭐⭐⭐
**목표:** 학생 정보를 엑셀에서 한 번에 등록

**구현 방식:**
```
엑셀 형식:
┌─────────┬──────┬────────┬─────┐
│ 학번    │ 이름 │ 반     │ 번호│
├─────────┼──────┼────────┼─────┤
│ 2024001 │ 김철수 │ 1반   │ 1   │
│ 2024002 │ 이영희 │ 1반   │ 2   │
│ 2024003 │ 박민준 │ 1반   │ 3   │
└─────────┴──────┴────────┴─────┘
```

**필요 라이브러리:**
- 프론트: `react-xlsx` 또는 `xlsx`
- 백: `xlsx` 또는 `exceljs`

**기능:**
- .xlsx 파일 선택 및 업로드
- 유효성 검사 (필수값 확인)
- 중복 체크
- 일괄 등록 또는 검토 후 등록
- 성공/실패 결과 리포트

**예상 소요 시간:** 3-4시간

---

### 3. **데이터 내보내기 (CSV/Excel)** ⭐⭐
**기능:**
- 평가 결과 → Excel로 다운로드
- 성장 분석 → Excel로 다운로드
- 학급 전체 통계 → Excel로 다운로드

**이점:**
- 학교 시스템과 연동 용이
- 보고서 작성 편함
- 학부모 통보서 작성 시 활용

## 🥈 **P2 (중요)** - 다음 단계 개선

### 4. **사용자 인증 (로그인 시스템)** ⭐⭐⭐
**필요성:**
- 교사별 학급 분리
- 데이터 보안
- 접근 제어

**구현:**

- 교사 계정 생성/관리
- 학급별 데이터 격리

---

### 5. **다중 실천 활동 유형 지원** ⭐⭐
**현황:** 현재 5가지 활동 유형만 고정

**개선:**
- 활동 유형을 관리자가 추가/수정 가능
- 평가 루브릭도 함께 커스터마이징
- 학교/교과별 맞춤형 구성

---

### 6. **고급 분석 대시보드** ⭐⭐⭐
**추가 통계:**
- 학생별 성장 궤적 시각화 (그래프)
- 반별 비교 분석
- 평가 항목별 점수 분포 (히스토그램)
- 시간대별 진도 현황
- 학생 그룹핑 (우수/주의 필요 등)

**기술:**
- `Chart.js` 또는 `Recharts` 활용

---

### 7. **평가 피드백 개선** ⭐⭐⭐
**현황:** Claude API 응답이 한정적

**개선:**
- 구체적인 강점/약점 분석
- 개선 방안 제시
- 다음 단계 학습 제안
- 교사가 피드백을 편집하고 학생에게 공유 가능
